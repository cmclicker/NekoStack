All phases

11 phases, each organized around a product or outcome. Re-order freely when priority shifts — the dependency graph constrains what must come before what, product priority is yours to set.

Phase 1 Foundation Closure Unlocks: stack self-governance 3 packages

Closes the Phase 0 gap. Every package built without lint and test accrues convention debt that compounds with each new addition.

Done when: any new package can be linted, type-checked, and tested using only NekoStack tooling — no per-package reinvention.
lint ⚡ test config
Phase 2 Governance Unlocks: policy enforcement in CI 1 package

Governance moves up from Phase 8. Encoding the product thesis as automated CI policy — not a markdown doc humans remember to consult. Every subsequent package is governed from day one.

Done when: a PR that violates the product thesis or skips the four-question gate causes CI to fail automatically.
governance
Phase 3 Observability Spine Unlocks: production-grade infrastructure 5 packages

Cross-cutting infrastructure every production-grade package needs. Build before any product goes to production.

Done when: any package can emit audit events, log structured records, surface errors, and expose a /health endpoint without scaffolding from scratch.
log errors health telemetry audit
