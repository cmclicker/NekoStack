  • schema-no-inline-zod — flags z.TYPE() calls outside schema files; 11 tests
  • prisma-json-cast — flags property access on Prisma.JsonValue typed vars without a cast; 12 tests
  • service-has-spec — flags *.service.ts files missing a co-located spec; 5 tests
  • controller-no-service-cycle — flags controller-to-controller imports; 10 tests
  • strict config — enables all five rules; exported as ./strict
  • 49 total tests (38 new + 11 from v0.1)
v0.3 Framework configs ✓ Shipped
  • react-no-inline-style — flags style={...} JSX attributes; use token-based classes instead; 10 tests
  • nest-no-orm-in-controller — flags ORM imports (Prisma/TypeORM/Mongoose/MikroORM) in controller files; 10 tests
  • react config — exported as ./react; curated NekoStack rule set for React 19 projects
  • nest config — exported as ./nest; full server-side rule set for NestJS 10+ projects
  • 69 total tests (20 new + 49 from v0.2)
v0.4 Security rules + first auto-fixer ✓ Shipped
  • no-hardcoded-secrets — flags secret-named variables assigned string literals + well-known credential prefixes (Stripe, GitHub, AWS, Google); test files exempt; 11 tests
  • no-raw-sql — flags unsafe Prisma raw-query methods; distinguishes safe tagged-template form from unsafe call form; 10 tests
  • schema-export-type — flags exported Zod schemas missing a z.infer<typeof schema> type alias; auto-fixer inserts the type export; 8 tests
  • react-no-dangerously-set-html — flags dangerouslySetInnerHTML prop (XSS risk); 7 tests
  • base config hardened: no-hardcoded-secrets added as universal security baseline
  • 105 total tests (36 new + 69 from v0.3)
v0.5 Module-boundary + quality rules ✓ Shipped
  • consistent-type-imports — flags import { Type } where import type { Type } is required; auto-fixer adds the type modifier
  • no-console-in-module — flags console.log / console.debug in non-test source files; permits console.error / console.warn
  • nest-event-handler-has-spec — extends service-has-spec to @EventPattern / @MessagePattern decorated methods
  • no-direct-date-now — flags Date.now() and new Date() outside test files; enforces injectable clock pattern
v0.6 Type-safety ✓ Shipped
  • no-type-assertion-to-any — flags x as any and <any>x; use unknown + type guard or the specific target type instead
  • no-non-null-assertion — flags postfix x!; use optional chaining, nullish coalescing, or an explicit null check
  • react-hook-naming — flags named functions that call React hooks but aren't named use* or PascalCase; anonymous callbacks exempt
  • nest-controller-response-type — flags @Get / @Post / … handler methods without an explicit return type
  • New ./recommended config — sensible defaults for any TypeScript project; layer ./react or ./nest on top
v1.0 Stable catalog ✓ Shipped
  • 19 rule docs (docs/rules/*.md) — Why, examples, options, when-to-disable for every shipped rule
  • docs/CONFIGS.md — hierarchy diagram + per-rule severity tables for all 5 configs
  • docs/MIGRATION_GUIDE.md — zero → base → recommended → strict upgrade path
  • @nekostack/schema linted with recommended on every CI run
