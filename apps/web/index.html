The core idea

One IR. Four outputs. Zero drift.

Every serious app defines the same shape in a TypeScript interface, a Zod schema, a JSON Schema, and an OpenAPI spec — and they drift. NekoStack normalizes one definition into a canonical intermediate representation, then generates each output from that IR. Where an output can't be faithful, it marks the gap instead of lying.

  • ✓ Strict-by-default validation with structured, normalized error codes
  • ✓ Deterministic, hash-tracked generated files — CI-checkable for staleness
  • ✓ No vendor lock-in: emit Zod 3 syntax on your schedule, not theirs
  • ✓ 1,294 tests on the schema package alone; public API frozen at v1.0
user.schema.ts 
import { s } from '@nekostack/schema';

export const User = s.object({
  id:          s.string().uuid(),
  email:       s.string().email(),
  displayName: s.string().min(1).max(50),
  createdAt:   s.isoDateTime(),
})
  .id("com.nekostack.auth.User")
  .version("1.0.0");

// → emits user.zod.ts, user.json,
//   user.openapi.json, user.d.ts
$ neko schema generate
The suite

Five packages, shipping today

Install the whole suite with nekostack, or pull scoped packages à la carte. Every one is published to npm under Apache-2.0.

@nekostack/schema

Canonical IR + multi-output schema system. Zod, JSON Schema, OpenAPI, and TypeScript from one definition.

npm i @nekostack/schema

@nekostack/cli

The neko command — schema validation, code generation, and migration management from one binary.

npm i -g @nekostack/cli

@nekostack/migrate-runner

A schema-data migration state machine that executes authored migrations against real records.

npm i @nekostack/migrate-runner

@nekostack/theme

W3C DTCG design-token pipeline. 3 themes × 2 modes, WCAG AA checked, emitted as CSS custom properties.

npm i @nekostack/theme

@nekostack/ui

92-component vanilla CSS library. Zero JS, zero React, token-pure — every color is a theme variable.

npm i @nekostack/ui

nekostack

The metapackage. One install pins version-locked, tested-together copies of all five primitives.

npm i nekostack

Quickstart

Up and running in a minute

Install the metapackage for everything, or just the scoped packages you need. zod is a peer dependency — add it when you run generated validators at runtime.

  • ✓ Pure ESM, TypeScript-native, no global config
  • ✓ Works in any Node project — framework-agnostic
  • ✓ neko CLI available via npx, no install required
terminal 
# everything, version-locked
$ npm install nekostack zod

# or à la carte
$ npm install @nekostack/schema
$ npm install @nekostack/theme @nekostack/ui

# run the CLI without installing
$ npx neko schema check "./schemas/**/*"
